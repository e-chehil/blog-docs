1.1 什么是共线性？

在统计建模（尤其是线性回归）中，共线性或多重共线性 是一个常见且棘手的问题。它指的是模型中的自变量（预测变量）之间存在高度相关关系。

1.2 为什么共线性是有害的？

想象一下，你想知道“学习时间”和“刷题数量”对“考试分数”的影响。但“学习时间”和“刷题数量”很可能是高度相关的（学得越久，刷题越多）。

当共线性发生时：

  1. 模型不稳定： 你的模型系数（比如“学习时间”的权重）会非常敏感。数据稍有变动，系数可能会发生剧烈变化，甚至正负颠倒。
  2. 难以解释： 你很难单独剥离出“学习时间”自身的贡献，因为它总是和“刷题数量”捆绑出现。
  3. 统计检验失效： 单个变量的 p 值可能会“膨胀”（变大），导致你错误地认为某个重要变量“不显著”。

为了构建一个稳健的模型，我们必须先诊断并处理共线性。本文将构建两个R函数，它们分别实现了两种共线性诊断方法：

  1. check_collinearity_cor：基于相关系数矩阵的贪心算法，用于自动移除高度相关的变量。
  2. check_collinearity_vif：基于方差膨胀因子（VIF），这是诊断共线性的“黄金标准”。

2 相关系数法

这种方法的核心思想是：如果两个变量的相关性太高（例如，绝对值超过0.7），我们应该扔掉其中一个。

2.1 何时使用？

  • 快速筛选： 当你有成百上千个变量，需要快速减少冗余时。
  • 非回归任务： 在非回归任务（如聚类、降维）前，去除冗余特征也很有用。
  • VIF 的前奏： 有时在计算 VIF 之前，用它先清理一遍，可以避免 VIF 计算时出现数学问题（如矩阵奇异）。

2.2 皮尔逊相关系数

这是最常见的相关性度量，通常用 \(r\) 表示。

\[ r_{xy} = \frac{\sum (x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum (x_i - \bar{x})^2 \sum (y_i - \bar{y})^2}} \]

  • \(r\) 的范围在 -1 到 +1 之间。
    • \(r = 1\)：完全正相关。
    • \(r = -1\)：完全负相关。
    • \(r = 0\)：完全不相关。

此函数使用 threshold（阈值）来判断 “太高” 的标准，它看的是相关性的绝对值 abs(r)。默认 threshold = 0.7 是一个常用的标准。

但如果A和B相关，B和C相关，我们该移除谁？check_collinearity_cor 实现了一个“贪心算法”来做这个决策：优先移除那个与“太多”其他变量高度相关的变量。

2.3 R 源码

check_collinearity_cor <- function(data, cols = NULL, threshold = 0.7) {
  # 1. 输入校验
  stopifnot(is.data.frame(data))

  # 2. 解析需要检查的列
  if (is.null(cols)) {
    cols_to_use <- names(data)
  } else if (is.numeric(cols)) {
    cols_to_use <- names(data)[cols]
  } else {
    cols_to_use <- as.character(cols)
  }
  if (any(!cols_to_use %in% names(data))) {
    stop("cols 中有不存在 data 的列名")
  }

  sub_data <- data[cols_to_use]

  # 3. 预处理：筛选数值型、处理缺失值和零方差
  is_num <- vapply(sub_data, is.numeric, logical(1))
  pred_df <- sub_data[is_num]
  if (ncol(pred_df) == 0) {
    warning("没有找到数值型列。")
    return(list(keep = character(0), removed = character(0)))
  }

  rows_ok <- stats::complete.cases(pred_df)
  pred_df_complete <- pred_df[rows_ok, , drop = FALSE]

  sds <- vapply(pred_df_complete, sd, numeric(1), na.rm = FALSE)
  is_zero_var <- (sds == 0) | is.na(sds)

  df_clean <- pred_df_complete[, !is_zero_var, drop = FALSE]

  # (发出警告)
  excluded_vars <- names(is_num)[!is_num]
  excluded_zero_vars <- names(is_zero_var)[is_zero_var]
  if (length(excluded_vars) > 0) {
    warning("已排除非数值型列: ", paste(excluded_vars, collapse = ", "))
  }
  if (length(excluded_zero_vars) > 0) {
    warning("已排除零方差/常数列: ", paste(excluded_zero_vars, collapse = ", "))
  }

  p <- ncol(df_clean)
  names_vec <- colnames(df_clean)

  if (p < 2) {
    warning("有效的数值型列不足 2 个，无需移除。")
    return(list(keep = names_vec, removed = character(0)))
  }

  # 4. 核心：计算相关矩阵和邻接矩阵
  cor_matrix <- stats::cor(df_clean)
  abs_cor <- abs(cor_matrix)
  diag(abs_cor) <- 0 # 对角线设为0，不关心自己和自己的相关性

  # adj 是一个逻辑矩阵，TRUE 表示相关性 "过高"
  adj <- abs_cor >= threshold
  diag(adj) <- FALSE

  # 5. 核心：计算"共线性负担"
  # neighbor_list 存储每个变量的 "高度相关邻居" 的索引
  neighbor_list <- lapply(seq_len(p), function(i) which(adj[i, ]))

  # cached_sum 存储每个变量的 "总相关负担"
  # 即：与所有 "高度相关邻居" 的相关系数绝对值之和
  cached_sum <- rowSums(abs_cor * adj)
  cached_sum[cached_sum == 0] <- NA_real_ # 没有邻居的，设为NA

  if (all(is.na(cached_sum))) {
    # 如果没有任何一对变量的相关性超过阈值
    return(list(keep = names_vec, removed = character(0)))
  }

  # 6. 核心：贪心算法迭代移除
  active <- setNames(rep(TRUE, p), names_vec) # 追踪哪些变量还 "存活"
  removed_order_idx <- integer(0) # 记录被移除的变量索引

  while (any(!is.na(cached_sum))) {
    # 6a. 找到 "最差" 的变量
    max_sum <- max(cached_sum, na.rm = TRUE)
    candidates_idx <- which(cached_sum == max_sum)

    # 6b. 处理平局 (Tie-Breaking)
    rem <- if (length(candidates_idx) == 1) {
      # 情况1：没有平局
      candidates_idx
    } else {
      # 情况2："总负担" 相同，看谁的 "单笔最大" 相关性更高
      max_cors <- vapply(candidates_idx, function(i) {
        neighs <- neighbor_list[[i]]
        if (length(neighs) > 0) max(abs_cor[i, neighs]) else -Inf
      }, numeric(1))

      max_max_cor <- max(max_cors)
      candidates_idx_2 <- candidates_idx[max_cors == max_max_cor]

      if (length(candidates_idx_2) == 1) {
        # 平局解开
        candidates_idx_2
      } else {
        # 情况3：如果 "单笔最大" 也相同，选择索引号最大的那个 (任意但确定的规则)
        candidates_idx_2[which.max(candidates_idx_2)]
      }
    }

    # 6c. "移除" 该变量
    active[rem] <- FALSE
    removed_order_idx <- c(removed_order_idx, rem)
    neighs_of_rem <- neighbor_list[[rem]]

    # 6d. 更新邻居信息
    cached_sum[rem] <- NA_real_ # 被移除的变量，"负担" 清零
    neighbor_list[[rem]] <- integer(0)

    if (length(neighs_of_rem) > 0) {
      for (k in neighs_of_rem) {
        if (active[k]) { # 仅更新还 "存活" 的邻居
          # 从邻居k的邻居列表中，移除 rem
          neighbor_list[[k]] <- neighbor_list[[k]][neighbor_list[[k]] != rem]

          if (!is.na(cached_sum[k])) {
            # 关键：更新邻居k的 "总负担"，减去来自 rem 的那部分
            cached_sum[k] <- cached_sum[k] - abs_cor[k, rem]
            if (length(neighbor_list[[k]]) == 0) {
              # 如果k也没有其他邻居了，"负担" 也清零
              cached_sum[k] <- NA_real_
            }
          }
        }
      }
    }
  }

  # 7. 返回结果
  list(
    keep = names_vec[active],
    removed = names_vec[removed_order_idx]
  )
}

2.4 步骤详解

  1. 输入校验 (1-2)：
  • stopifnot(is.data.frame(data))：确保输入是数据框。
  • cols 解析：允许用户通过列名（字符）、索引（数值）或 NULL（所有列）来指定要检查的变量。
  1. 数据预处理 (3)：这是至关重要的一步。
  • vapply(sub_data, is.numeric, ...)：使用 vapply（一个比 sapply 更安全、更快的版本）检查哪些列是数值型。相关性只能在数值型变量上计算。
  • stats::complete.cases(pred_df)：找出完整行。相关系数矩阵通常在“成对完整观测”（pairwise-complete）或“列表完整观测”（listwise-complete）上计算。这里采用后者（stats::cor 的默认行为），即只使用在所有数值列上都没有NA的行。
  • vapply(..., sd, ...)：计算标准差。
  • is_zero_var：检测零方差列（即常数列）。常数列的标准差为0（或因全是NA而导致 sd 为 NA）。这些列必须被移除，因为它们在相关性计算中没有意义，并且会导致 stats::cor 报错。
  1. 计算相关矩阵 (4)：
  • cor_matrix <- stats::cor(df_clean)：计算皮尔逊（Pearson）相关系数矩阵 \(R\)。
  • abs_cor <- abs(cor_matrix)：取绝对值，因为我们关心的是相关性的大小，而不是方向（正或负）。
  • adj <- abs_cor >= threshold：这是一个邻接矩阵（Adjacency Matrix）。在图论中，如果两个变量（节点）的相关性超过阈值，我们就说它们之间有一条“边”。adj[i, j] == TRUE 表示 \(i\) 和 \(j\) 是“高度相关”的。
  1. 计算“共线性负担” (5)：
  • 这是该算法的核心启发式。
  • neighbor_list：存储了图的结构。neighbor_list[[i]] 包含了所有与 \(i\) 高度相关的邻居。
  • cached_sum <- rowSums(abs_cor * adj)：这是一个巧妙的向量化计算。abs_cor * adj 的结果是：如果 \(i, j\) 不高度相关，adj 中的 FALSE 会使乘积为0；如果相关，adj 中的 TRUE (转为1) 会保留 abs_cor[i, j] 的值。因此，rowSums 计算的就是每个变量 \(i\) 的“总相关负担”： \[ \text{Sum}_i = \sum_{j \neq i} |R_{ij}| \times I(|R_{ij}| \ge \tau) \] 其中 \(\tau\) (tau) 是 threshold，\(I\) 是指示函数。
  • cached_sum[cached_sum == 0] <- NA_real_：将没有“邻居”的变量（即与任何其他变量的相关性都不超过阈值）设为 NA，表示它们是安全的，不参与后续的移除迭代。
  1. 贪心算法迭代 (6)：
  • while (any(!is.na(cached_sum)))：只要还有变量的“负担”不是 NA（即还存在高相关性），就继续循环。
  • (6a) 找到最差的变量：max_sum <- max(cached_sum, ...)。我们找到那个具有最高“总相关负担”的变量，它被认为是“共线性问题最严重的”。
  • (6b) 处理平局：
    • 如果多个变量的“总负担”一样高（candidates_idx），我们引入第二条规则：看谁的“最大单笔相关”最高。例如，A的总负担是1.5 (0.8 + 0.7)，B的总负担也是1.5 (0.75 + 0.75)。A的单笔最大是0.8，B的是0.75，我们会优先移除A。
    • 如果“最大单笔相关”也一样（candidates_idx_2），我们引入第三条规则：选择索引号最大的那个。这只是为了保证算法的确定性（每次运行结果都一样），没有特殊的统计学含义。
    • (6c) 移除变量：active[rem] <- FALSE，并记录到 removed_order_idx。
    • (6d) 更新邻居：这是算法最高效的部分。当变量 rem 被移除后，我们不需要重新计算整个相关矩阵。我们只需要更新 rem 的邻居 k：
      • neighbor_list[[k]]：从 k 的邻居中移除 rem。
      • cached_sum[k] - abs_cor[...]：k 的“总负担”减轻了，因为它不再需要对 rem 的相关性“负责”。
  1. 返回结果 (7)：
  • names_vec[active]：active 向量中仍为 TRUE 的变量，即“存活”下来的。
  • names_vec[removed_order_idx]：按移除顺序列出被移除的变量。

3 方差膨胀因子法

VIF 是迄今为止最流行和最受推荐的共线性诊断工具。

3.1 什么是VIF？

方差膨胀因子（VIF） 衡量的是一个自变量 \(X_j\) 的方差，在存在其他自变量时，相对于不存在其他自变量时，膨胀了多少倍。它不像相关系数那样只看 “两两” 关系，它衡量的是一个自变量在多大程度上可以被所有其他自变量来解释。

计算 \(VIF_j\)（第 \(j\) 个变量的 VIF）的步骤：

  1. 建立“辅助回归”： 假装第 \(j\) 个变量 \(X_j\) 是因变量，用所有其他自变量（\(X_1, X_2, ..., X_{j-1}, X_{j+1}, ...\)）来预测它。 \[ \begin{split} X_j = \beta_0 + \beta_1 X_1 + \dots + \beta_{j-1} X_{j-1} \\ \quad +\, \beta_{j+1} X_{j+1} + \dots + \text{error} \end{split} \]
  2. 获取 \(R_j^2\)： 查看这个辅助回归模型的 \(R^2\)（R-squared，决定系数）。这个 \(R_j^2\) 衡量了其他变量对 \(X_j\) 的解释力度。
    • 如果 \(R_j^2 = 0\)，说明 \(X_j\) 与其他变量完全无关。
    • 如果 \(R_j^2\) 接近 1，说明 \(X_j\) 几乎可以被其他变量完美预测（即存在严重共线性）。
  3. 计算 VIF： VIF 的公式非常简单： \[ VIF_j = \frac{1}{1 - R_j^2} \]

如何解读 VIF 值？

  • \(VIF = 1\)： 当 \(R_j^2 = 0\) 时。这是最理想的情况，表示该变量与模型中其他自变量完全不相关。
  • \(VIF = 5\) 或 \(10\)： 这是常用的判断阈值（threshold 参数）。
    • \(VIF = 10\) 对应 \(R_j^2 = 0.9\)（即该变量 90% 的方差能被其他变量解释）。
    • \(VIF = 5\) 对应 \(R_j^2 = 0.8\)。
  • \(VIF \to \infty\)（无穷大）： 当 \(R_j^2 = 1\) 时（完全共线性）。这意味着 \(X_j\) 是其他变量的完美线性组合（例如 \(X_j = 2 \times X_k\)）。

VIF 的含义：VIF 告诉我们，由于 \(X_j\) 与其他自变量的相关性，导致 \(X_j\) 的回归系数的方差比理想情况（VIF=1）下膨胀了多少倍。VIF 越高，系数越不稳定。

3.2 VIF 的两种计算方法

check_collinearity_vif 函数非常智能，它混合了两种计算方法：

  1. 方法 A（慢速，但直观）：辅助回归法
  • 如上所述，对 \(p\) 个变量中的每一个，都运行一次辅助回归，总共 \(p\) 次回归。
  • 优点：概念清晰，易于理解，逐个计算，鲁棒性强。
  • 缺点：当 \(p\) 很大时，计算量巨大（\(p\) 次回归）。
  1. 方法 B（快速，但有前提）：相关矩阵求逆法
  • 数学定理：所有变量的 VIF 值，可以一次性从相关矩阵 \(R\) 的逆 \(R^{-1}\) 中得到！
  • 具体来说，VIF 向量等于 \(R^{-1}\) 的对角线元素。 \[ \text{VIFs} = \text{diag}(R^{-1}) \]
    • 优点：极其高效。只需计算一次相关矩阵和一次矩阵求逆，就能得到所有 \(p\) 个VIF。
    • 缺点：有前提条件！
      • 相关矩阵 \(R\) 必须是可逆的（非奇异的）。
      • 如果存在完全共线性（\(R_j^2 = 1\)），\(R\) 将不可逆。
      • 如果 \(p \ge n\)（变量数多于观测数），\(R\) 也不可逆。
      • 如果矩阵“病态”（ill-conditioned，即接近不可逆），\(R^{-1}\) 的计算会充满数值误差。

check_collinearity_vif 的策略是：默认尝试方法B（快速），如果检测到任何问题，则自动回退到方法A（慢速）。

check_collinearity_vif <- function(data, cols = NULL, threshold = 10) {
  # 1. 输入校验和预处理 (与 check_collinearity_cor 类似)
  stopifnot(is.data.frame(data))
  rcond_tol <- .Machine$double.eps^0.5 # 用于检测"病态矩阵"的极小阈值

  # (解析 cols ...)
  if (is.null(cols)) {
    cols_to_use <- names(data)
  } else if (is.numeric(cols)) {
    cols_to_use <- names(data)[cols]
  } else {
    cols_to_use <- as.character(cols)
  }
  if (any(!cols_to_use %in% names(data))) {
    stop("cols 中有不存在 data 的列名")
  }

  if (length(cols_to_use) == 0) {
    warning("没有指定任何列。")
    return(data.frame(predictor = character(0), vif = numeric(0), is_high = logical(0)))
  }

  sub_data <- data[cols_to_use]

  # (筛选数值型...)
  is_num <- vapply(sub_data, is.numeric, logical(1))
  valid_cols_initial <- names(is_num)[is_num]
  excluded_non_numeric <- names(is_num)[!is_num]
  if (length(excluded_non_numeric) > 0) {
    warning("已排除非数值型列: ", paste(excluded_non_numeric, collapse = ", "))
  }
  if (length(valid_cols_initial) == 0) {
    warning("所有指定的列都不是数值型，无法计算 VIF。")
    return(data.frame(predictor = character(0), vif = numeric(0), is_high = logical(0)))
  }

  pred_df <- sub_data[valid_cols_initial]

  # (处理缺失值...)
  rows_ok <- stats::complete.cases(pred_df)
  pred_df_complete <- pred_df[rows_ok, , drop = FALSE]
  n_obs <- nrow(pred_df_complete)

  # 初始化VIF向量，默认为 NA
  vif_values <- setNames(
    rep(NA_real_, length(valid_cols_initial)),
    valid_cols_initial
  )

  if (n_obs < 2) {
    warning(paste("完整观测行数 (n_obs =", n_obs, ") 不足 2，无法计算 VIF。"))
  } else {
    # (处理零方差列...)
    sds <- vapply(pred_df_complete, sd, numeric(1), na.rm = FALSE)
    is_zero_var <- (sds == 0) | is.na(sds)
    excluded_zero_var <- names(is_zero_var)[is_zero_var]
    valid_cols_final <- names(is_zero_var)[!is_zero_var]

    if (length(excluded_zero_var) > 0) {
      warning("已排除零方差/常数列 (VIF 设为 NA): ", paste(excluded_zero_var, collapse = ", "))
    }

    p <- length(valid_cols_final)

    if (p == 0) {
      warning("所有数值型列（在完整行上）都是常数或NA，VIF 均设为 NA。")
    } else if (p == 1) {
      # 只有一个变量，没有共线性，VIF=1
      vif_values[valid_cols_final] <- 1.0
    } else {
      # 2. 核心：尝试 "方法B" (快速)，并设置回退标志
      use_lm_method <- FALSE

      # 2a. 回退检查 1: p >= n
      if (p >= n_obs) {
        warning(paste("p (有效自变量数 =", p, ") >= n (观测行数 =", n_obs, ")。相关矩阵奇异，将回退到 lm 辅助回归法。"))
        use_lm_method <- TRUE
      }

      if (!use_lm_method) {
        R <- stats::cor(pred_df_complete[valid_cols_final]) # nolint

        # 2b. 回退检查 2: 矩阵是否正定 (Positive Definite)
        R_chol <- try(chol(R), silent = TRUE) # nolint

        if (inherits(R_chol, "try-error")) {
          warning("相关矩阵非正定，自动回退到逐变量辅助回归法。")
          use_lm_method <- TRUE
        } else {
          # 2c. 回退检查 3: 矩阵是否 "病态" (Ill-Conditioned)
          r_cond <- rcond(R)

          if (r_cond < rcond_tol) {
            warning(sprintf("相关矩阵病态 (rcond = %g < %g)，自动回退到逐变量辅助回归法。", r_cond, rcond_tol))
            use_lm_method <- TRUE
          } else {
            # 2d. 成功！使用 "方法B"
            inv_r <- chol2inv(R_chol)
            vif_values[valid_cols_final] <- diag(inv_r)
          }
        }
      }

      # 3. 核心：执行 "方法A" (慢速回退)
      if (use_lm_method) {
        # 准备好设计矩阵 (加一列截距项，便于 lm.fit)
        X_all_with_intercept <- cbind( # nolint
          `(Intercept)` = 1,
          as.matrix(pred_df_complete[valid_cols_final])
        )

        for (i in seq_len(p)) {
          this_pred_name <- valid_cols_final[i]

          # 准备 y (即 X_j) 和 X (即其他变量)
          y <- X_all_with_intercept[, i + 1, drop = FALSE]
          X_matrix <- X_all_with_intercept[, -(i + 1), drop = FALSE] # nolint

          # 使用底层的 lm.fit (比 lm() 快)
          fit <- try(lm.fit(x = X_matrix, y = y), silent = TRUE)

          if (inherits(fit, "try-error")) {
            r2 <- NA_real_
          } else {
            # 手动计算 R-squared
            rss <- sum(fit$residuals^2)
            # 注意：TSS (Total Sum of Squares) 的计算
            tss <- sum(y^2) - sum(y)^2 / n_obs

            if (abs(tss) < .Machine$double.eps) {
              # 如果 y 是常数 (本应被零方差检查捕获，但作为双重保险)
              r2 <- NA_real_
            } else {
              r2 <- 1 - rss / tss
            }

            # 修复浮点数精度问题 (e.g., 1.0000000000000002)
            if (!is.na(r2) && r2 > 1 && (r2 - 1) < .Machine$double.eps) {
              r2 <- 1.0
            }
          }

          # 3b. 从 R^2 计算 VIF
          if (is.na(r2)) {
            vif_val <- NA_real_
          } else if (abs(1 - r2) < .Machine$double.eps) {
            # R^2 = 1 (完全共线性)
            vif_val <- Inf
          } else {
            vif_val <- 1 / (1 - r2)
          }
          vif_values[this_pred_name] <- vif_val
        }
      }
    }
  }

  # 4. 格式化输出
  vif_table <- data.frame(
    predictor = names(vif_values),
    vif = vif_values,
    is_high = !is.na(vif_values) & vif_values >= threshold
  )
  rownames(vif_table) <- NULL

  # 按VIF值降序排列，NA值放在最后
  vif_table <- vif_table[
    order(vif_table$vif, decreasing = TRUE, na.last = TRUE),
  ]

  return(vif_table)
}

3.4 步骤详解

  1. 预处理 (1)：
  • 与 check_collinearity_cor 几乎相同：筛选数值列、complete.cases、排除零方差列。
  • vif_values 被初始化为 NA。对于被排除的非数值列或零方差列，它们最终的VIF将保持为 NA。
  • 特殊情况：如果 \(p=1\)（只有一个有效变量），VIF 直接设为 1。
  1. 核心逻辑 - 尝试方法B (2)：
  • use_lm_method 标志：这是一个“开关”，默认为 FALSE（使用方法B），如果任何检查失败，则设为 TRUE（切换到方法A）。
  • (2a) 检查 p >= n：这是最常见的奇点（Singularity）来源。如果变量多于观测，矩阵 \(X'X\)（相关矩阵 \(R\) 与此相关）是不可逆的。必须回退。
  • (2b) 检查正定性：R_chol <- try(chol(R), ...)。chol 是 Cholesky 分解，是计算矩阵逆的第一步，也是检查矩阵是否为正定矩阵（Positive Definite）的最快方法。如果 chol 失败（例如，由于浮点数不精确导致对角线出现非正值），try 会捕获错误，我们必须回退。
  • (2c) 检查病态：r_cond <- rcond(R)。rcond 计算“条件数（Condition Number）的倒数”。
    • 条件数是衡量“矩阵接近不可逆”的指标。一个大的条件数（rcond 相应地会很小）意味着矩阵是“病态”的。
    • 对病态矩阵求逆，会导致极大的数值误差。例如，\(1 / (1 - 0.9999999999999999)\) 应该是一个大数，但 \(1 / (1 - 1.0000000000000001)\) 却是一个负数！
    • rcond_tol 是一个常用的阈值（约 \(1.49 \times 10^{-8}\)）。如果 rcond(R) 小于此值，我们认为求逆是不安全的，必须回退。
  • (2d) 成功：如果所有检查都通过，inv_r <- chol2inv(R_chol)（从Cholesky分解高效计算逆）并 diag(inv_r)（提取对角线）一步到位。
  1. 核心逻辑 - 执行方法A (3)：
  • if (use_lm_method)：如果任何一个检查失败，此开关为 TRUE，执行此代码块。
  • for (i in seq_len(p))：对每一个变量 \(X_j\) (即 this_pred_name) 执行循环。
  • y <- ... 和 X_matrix <- ...：巧妙地从总矩阵中提取出 \(X_j\) 作为 y，提取出 \(X_{-j}\)（所有其他变量）作为 X_matrix。
  • fit <- try(lm.fit(...), ...)：
    • lm.fit 是R中执行回归的底层函数。它跳过了 lm() 函数中涉及的公式解析、数据框转换等耗时步骤，直接在矩阵 x 和向量 y 上进行计算，速度快得多。
  • 手动计算 \(R^2\)：
    • rss <- sum(fit$residuals^2)：残差平方和（Residual Sum of Squares）。
    • tss <- sum(y^2) - sum(y)^2 / n_obs：总平方和（Total Sum of Squares）。
    • r2 <- 1 - rss / tss：\(R^2\) 的标准定义。
  • VIF 计算：
    • if (abs(1 - r2) < ...)：这是对 \(R^2 = 1\) 的稳健检查（考虑到浮点精度）。
    • vif_val <- Inf：如果 \(R^2 = 1\)，则 \(1 - R^2 = 0\)，VIF 为无穷大（Inf）。
    • vif_val <- 1 / (1 - r2)：标准公式。
  1. 格式化输出 (4)：
  • 将 vif_values 向量转换为一个整洁的 data.frame。
  • order(vif_table$vif, ...)：一个非常有用的排序，将VIF值从高到低排列，并将所有 NA 值（来自非数值型或零方差列）放在最后。

4 结论

我们深入分析了两种诊断共线性的R函数：

  1. check_collinearity_cor：
  • 做什么：通过贪心算法自动移除变量。
  • 原理：计算一个“总相关负担” (cached_sum)，并迭代地移除“负担”最重的变量，同时高效地更新其邻居的“负担”。
  • 适用场景：适用于特征工程和预筛选，当你希望快速减少特征数量并消除成对高相关性时。
  1. check_collinearity_vif：
  • 做什么：计算每个变量的VIF值，用于诊断。
  • 原理：实现了两种方法。首选 \(VIFs = \text{diag}(R^{-1})\)（快速，矩阵求逆），但在检测到数值不稳定（p >= n、非正定、病态）时，自动回退到 \(VIF_j = 1 / (1 - R_j^2)\)（慢速，辅助回归）。
  • 适用场景：这是标准的共线性诊断方法。它比相关矩阵法更强大，因为它可以检测到一个变量被多个其他变量组合解释的情况（例如，\(X_1 \approx 0.5 X_2 + 0.5 X_3\)，即使 \(X_1\) 与 \(X_2\)、\(X_3\) 的两两相关性都不高）。

## 导言

### 什么是共线性？

在统计建模（尤其是线性回归）中，**共线性**或**多重共线性** 是一个常见且棘手的问题。它指的是模型中的自变量（预测变量）之间存在高度相关关系。

### 为什么共线性是有害的？

想象一下，你想知道“学习时间”和“刷题数量”对“考试分数”的影响。但“学习时间”和“刷题数量”很可能是高度相关的（学得越久，刷题越多）。

当共线性发生时：

1.  **模型不稳定：** 你的模型系数（比如“学习时间”的权重）会非常敏感。数据稍有变动，系数可能会发生剧烈变化，甚至正负颠倒。
2.  **难以解释：** 你很难单独剥离出“学习时间”**自身**的贡献，因为它总是和“刷题数量”捆绑出现。
3.  **统计检验失效：** 单个变量的 p 值可能会“膨胀”（变大），导致你错误地认为某个重要变量“不显著”。

为了构建一个稳健的模型，我们必须先诊断并处理共线性。本文将构建两个R函数，它们分别实现了两种共线性诊断方法：

1.  **`check_collinearity_cor`**：基于**相关系数矩阵**的贪心算法，用于**自动移除**高度相关的变量。
2.  **`check_collinearity_vif`**：基于**方差膨胀因子（VIF）**，这是诊断共线性的“黄金标准”。

## 相关系数法

这种方法的核心思想是：**如果两个变量的相关性太高（例如，绝对值超过0.7），我们应该扔掉其中一个。**

### 何时使用？

-   **快速筛选：** 当你有成百上千个变量，需要快速减少冗余时。
-   **非回归任务：** 在非回归任务（如聚类、降维）前，去除冗余特征也很有用。
-   **VIF 的前奏：** 有时在计算 VIF 之前，用它先清理一遍，可以避免 VIF 计算时出现数学问题（如矩阵奇异）。

### 皮尔逊相关系数

这是最常见的相关性度量，通常用 $r$ 表示。

$$
r_{xy} = \frac{\sum (x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum (x_i - \bar{x})^2 \sum (y_i - \bar{y})^2}}
$$

-   $r$ 的范围在 -1 到 +1 之间。
    -   $r = 1$：完全正相关。
    -   $r = -1$：完全负相关。
    -   $r = 0$：完全不相关。

此函数使用 `threshold`（阈值）来判断 "太高" 的标准，它看的是相关性的**绝对值** `abs(r)`。默认 `threshold = 0.7` 是一个常用的标准。

但如果A和B相关，B和C相关，我们该移除谁？`check_collinearity_cor` 实现了一个“贪心算法”来做这个决策：**优先移除那个与“太多”其他变量高度相关的变量。**

```{r}
check_collinearity_cor <- function(data, cols = NULL, threshold = 0.7) {
  # 1. 输入校验
  stopifnot(is.data.frame(data))

  # 2. 解析需要检查的列
  if (is.null(cols)) {
    cols_to_use <- names(data)
  } else if (is.numeric(cols)) {
    cols_to_use <- names(data)[cols]
  } else {
    cols_to_use <- as.character(cols)
  }
  if (any(!cols_to_use %in% names(data))) {
    stop("cols 中有不存在 data 的列名")
  }

  sub_data <- data[cols_to_use]

  # 3. 预处理：筛选数值型、处理缺失值和零方差
  is_num <- vapply(sub_data, is.numeric, logical(1))
  pred_df <- sub_data[is_num]
  if (ncol(pred_df) == 0) {
    warning("没有找到数值型列。")
    return(list(keep = character(0), removed = character(0)))
  }

  rows_ok <- stats::complete.cases(pred_df)
  pred_df_complete <- pred_df[rows_ok, , drop = FALSE]

  sds <- vapply(pred_df_complete, sd, numeric(1), na.rm = FALSE)
  is_zero_var <- (sds == 0) | is.na(sds)

  df_clean <- pred_df_complete[, !is_zero_var, drop = FALSE]

  # (发出警告)
  excluded_vars <- names(is_num)[!is_num]
  excluded_zero_vars <- names(is_zero_var)[is_zero_var]
  if (length(excluded_vars) > 0) {
    warning("已排除非数值型列: ", paste(excluded_vars, collapse = ", "))
  }
  if (length(excluded_zero_vars) > 0) {
    warning("已排除零方差/常数列: ", paste(excluded_zero_vars, collapse = ", "))
  }

  p <- ncol(df_clean)
  names_vec <- colnames(df_clean)

  if (p < 2) {
    warning("有效的数值型列不足 2 个，无需移除。")
    return(list(keep = names_vec, removed = character(0)))
  }

  # 4. 核心：计算相关矩阵和邻接矩阵
  cor_matrix <- stats::cor(df_clean)
  abs_cor <- abs(cor_matrix)
  diag(abs_cor) <- 0 # 对角线设为0，不关心自己和自己的相关性

  # adj 是一个逻辑矩阵，TRUE 表示相关性 "过高"
  adj <- abs_cor >= threshold
  diag(adj) <- FALSE

  # 5. 核心：计算"共线性负担"
  # neighbor_list 存储每个变量的 "高度相关邻居" 的索引
  neighbor_list <- lapply(seq_len(p), function(i) which(adj[i, ]))

  # cached_sum 存储每个变量的 "总相关负担"
  # 即：与所有 "高度相关邻居" 的相关系数绝对值之和
  cached_sum <- rowSums(abs_cor * adj)
  cached_sum[cached_sum == 0] <- NA_real_ # 没有邻居的，设为NA

  if (all(is.na(cached_sum))) {
    # 如果没有任何一对变量的相关性超过阈值
    return(list(keep = names_vec, removed = character(0)))
  }

  # 6. 核心：贪心算法迭代移除
  active <- setNames(rep(TRUE, p), names_vec) # 追踪哪些变量还 "存活"
  removed_order_idx <- integer(0) # 记录被移除的变量索引

  while (any(!is.na(cached_sum))) {
    # 6a. 找到 "最差" 的变量
    max_sum <- max(cached_sum, na.rm = TRUE)
    candidates_idx <- which(cached_sum == max_sum)

    # 6b. 处理平局 (Tie-Breaking)
    rem <- if (length(candidates_idx) == 1) {
      # 情况1：没有平局
      candidates_idx
    } else {
      # 情况2："总负担" 相同，看谁的 "单笔最大" 相关性更高
      max_cors <- vapply(candidates_idx, function(i) {
        neighs <- neighbor_list[[i]]
        if (length(neighs) > 0) max(abs_cor[i, neighs]) else -Inf
      }, numeric(1))

      max_max_cor <- max(max_cors)
      candidates_idx_2 <- candidates_idx[max_cors == max_max_cor]

      if (length(candidates_idx_2) == 1) {
        # 平局解开
        candidates_idx_2
      } else {
        # 情况3：如果 "单笔最大" 也相同，选择索引号最大的那个 (任意但确定的规则)
        candidates_idx_2[which.max(candidates_idx_2)]
      }
    }

    # 6c. "移除" 该变量
    active[rem] <- FALSE
    removed_order_idx <- c(removed_order_idx, rem)
    neighs_of_rem <- neighbor_list[[rem]]

    # 6d. 更新邻居信息
    cached_sum[rem] <- NA_real_ # 被移除的变量，"负担" 清零
    neighbor_list[[rem]] <- integer(0)

    if (length(neighs_of_rem) > 0) {
      for (k in neighs_of_rem) {
        if (active[k]) { # 仅更新还 "存活" 的邻居
          # 从邻居k的邻居列表中，移除 rem
          neighbor_list[[k]] <- neighbor_list[[k]][neighbor_list[[k]] != rem]

          if (!is.na(cached_sum[k])) {
            # 关键：更新邻居k的 "总负担"，减去来自 rem 的那部分
            cached_sum[k] <- cached_sum[k] - abs_cor[k, rem]
            if (length(neighbor_list[[k]]) == 0) {
              # 如果k也没有其他邻居了，"负担" 也清零
              cached_sum[k] <- NA_real_
            }
          }
        }
      }
    }
  }

  # 7. 返回结果
  list(
    keep = names_vec[active],
    removed = names_vec[removed_order_idx]
  )
}
```

### 步骤详解

1.  **输入校验 (1-2)**：

-   `stopifnot(is.data.frame(data))`：确保输入是数据框。
-   `cols` 解析：允许用户通过**列名**（字符）、**索引**（数值）或 `NULL`（所有列）来指定要检查的变量。

2.  **数据预处理 (3)**：这是至关重要的一步。

-   `vapply(sub_data, is.numeric, ...)`：使用 `vapply`（一个比 `sapply` 更安全、更快的版本）检查哪些列是**数值型**。相关性只能在数值型变量上计算。
-   `stats::complete.cases(pred_df)`：找出**完整行**。相关系数矩阵通常在“成对完整观测”（pairwise-complete）或“列表完整观测”（listwise-complete）上计算。这里采用后者（`stats::cor` 的默认行为），即只使用在*所有*数值列上都没有NA的行。
-   `vapply(..., sd, ...)`：计算标准差。
-   `is_zero_var`：检测**零方差列**（即常数列）。常数列的标准差为0（或因全是NA而导致 `sd` 为 `NA`）。这些列必须被移除，因为它们在相关性计算中没有意义，并且会导致 `stats::cor` 报错。

1.  **计算相关矩阵 (4)**：

-   `cor_matrix <- stats::cor(df_clean)`：计算皮尔逊（Pearson）相关系数矩阵 $R$。
-   `abs_cor <- abs(cor_matrix)`：取绝对值，因为我们关心的是相关性的大小，而不是方向（正或负）。
-   `adj <- abs_cor >= threshold`：这是一个**邻接矩阵**（Adjacency Matrix）。在图论中，如果两个变量（节点）的相关性超过阈值，我们就说它们之间有一条“边”。`adj[i, j] == TRUE` 表示 $i$ 和 $j$ 是“高度相关”的。

4.  **计算“共线性负担” (5)**：

-   这是该算法的**核心启发式**。
-   `neighbor_list`：存储了图的结构。`neighbor_list[[i]]` 包含了所有与 $i$ 高度相关的邻居。
-   `cached_sum <- rowSums(abs_cor * adj)`：这是一个巧妙的向量化计算。`abs_cor * adj` 的结果是：如果 $i, j$ 不高度相关，`adj` 中的 `FALSE` 会使乘积为0；如果相关，`adj` 中的 `TRUE` (转为1) 会保留 `abs_cor[i, j]` 的值。因此，`rowSums` 计算的就是每个变量 $i$ 的“总相关负担”： $$
    \text{Sum}_i = \sum_{j \neq i} |R_{ij}| \times I(|R_{ij}| \ge \tau)
    $$ 其中 $\tau$ (tau) 是 `threshold`，$I$ 是指示函数。
-   `cached_sum[cached_sum == 0] <- NA_real_`：将没有“邻居”的变量（即与任何其他变量的相关性都不超过阈值）设为 `NA`，表示它们是安全的，不参与后续的移除迭代。

5.  **贪心算法迭代 (6)**：

-   `while (any(!is.na(cached_sum)))`：只要还有变量的“负担”不是 `NA`（即还存在高相关性），就继续循环。
-   **(6a) 找到最差的变量**：`max_sum <- max(cached_sum, ...)`。我们找到那个具有最高“总相关负担”的变量，它被认为是“共线性问题最严重的”。
-   **(6b) 处理平局**：
    -   如果多个变量的“总负担”一样高（`candidates_idx`），我们引入第二条规则：**看谁的“最大单笔相关”最高**。例如，A的总负担是1.5 (0.8 + 0.7)，B的总负担也是1.5 (0.75 + 0.75)。A的单笔最大是0.8，B的是0.75，我们会优先移除A。
    -   如果“最大单笔相关”也一样（`candidates_idx_2`），我们引入第三条规则：**选择索引号最大的那个**。这只是为了保证算法的*确定性*（每次运行结果都一样），没有特殊的统计学含义。
    -   **(6c) 移除变量**：`active[rem] <- FALSE`，并记录到 `removed_order_idx`。
    -   **(6d) 更新邻居**：这是算法最高效的部分。当变量 `rem` 被移除后，我们**不需要**重新计算整个相关矩阵。我们只需要更新 `rem` 的邻居 `k`：
        -   `neighbor_list[[k]]`：从 `k` 的邻居中移除 `rem`。
        -   `cached_sum[k] - abs_cor[...]`：`k` 的“总负担”减轻了，因为它不再需要对 `rem` 的相关性“负责”。

6.  **返回结果 (7)**：

-   `names_vec[active]`：`active` 向量中仍为 `TRUE` 的变量，即“存活”下来的。
-   `names_vec[removed_order_idx]`：按移除顺序列出被移除的变量。

## 方差膨胀因子法

VIF 是迄今为止最流行和最受推荐的共线性诊断工具。

### 什么是VIF？

**方差膨胀因子（VIF）** 衡量的是一个自变量 $X_j$ 的方差，在*存在*其他自变量时，相对于*不存在*其他自变量时，膨胀了多少倍。它不像相关系数那样只看 "两两" 关系，它衡量的是**一个自变量在多大程度上可以被所有其他自变量来解释**。

计算 $VIF_j$（第 $j$ 个变量的 VIF）的步骤：

1.  **建立“辅助回归”**： 假装第 $j$ 个变量 $X_j$ 是因变量，用**所有其他**自变量（$X_1, X_2, ..., X_{j-1}, X_{j+1}, ...$）来预测它。 $$
      \begin{split}
      X_j = \beta_0 + \beta_1 X_1 + \dots + \beta_{j-1} X_{j-1} \\
      \quad +\, \beta_{j+1} X_{j+1} + \dots + \text{error}
      \end{split}
      $$
2.  **获取** $R_j^2$： 查看这个辅助回归模型的 $R^2$（R-squared，决定系数）。这个 $R_j^2$ 衡量了其他变量对 $X_j$ 的解释力度。
    -   如果 $R_j^2 = 0$，说明 $X_j$ 与其他变量完全无关。
    -   如果 $R_j^2$ 接近 1，说明 $X_j$ 几乎可以被其他变量完美预测（即存在严重共线性）。
3.  **计算 VIF**： VIF 的公式非常简单： $$
      VIF_j = \frac{1}{1 - R_j^2}
      $$

**如何解读 VIF 值？**

-   $VIF = 1$： 当 $R_j^2 = 0$ 时。这是最理想的情况，表示该变量与模型中其他自变量**完全不相关**。
-   $VIF = 5$ 或 $10$： 这是常用的判断阈值（`threshold` 参数）。
    -   $VIF = 10$ 对应 $R_j^2 = 0.9$（即该变量 90% 的方差能被其他变量解释）。
    -   $VIF = 5$ 对应 $R_j^2 = 0.8$。
-   $VIF \to \infty$（无穷大）： 当 $R_j^2 = 1$ 时（完全共线性）。这意味着 $X_j$ 是其他变量的完美线性组合（例如 $X_j = 2 \times X_k$）。

**VIF 的含义**：VIF 告诉我们，由于 $X_j$ 与其他自变量的相关性，导致 $X_j$ 的回归系数的**方差**比理想情况（VIF=1）下**膨胀了多少倍**。VIF 越高，系数越不稳定。

### VIF 的两种计算方法

`check_collinearity_vif` 函数非常智能，它混合了两种计算方法：

1.  **方法 A（慢速，但直观）：辅助回归法**

-   如上所述，对 $p$ 个变量中的每一个，都运行一次辅助回归，总共 $p$ 次回归。
-   **优点**：概念清晰，易于理解，逐个计算，鲁棒性强。
-   **缺点**：当 $p$ 很大时，计算量巨大（$p$ 次回归）。

2.  **方法 B（快速，但有前提）：相关矩阵求逆法**

-   **数学定理**：所有变量的 VIF 值，可以一次性从**相关矩阵** $R$ 的逆 $R^{-1}$ 中得到！
-   具体来说，**VIF 向量等于** $R^{-1}$ 的对角线元素。 $$
    \text{VIFs} = \text{diag}(R^{-1})
    $$
    -   **优点**：极其高效。只需计算一次相关矩阵和一次矩阵求逆，就能得到所有 $p$ 个VIF。
    -   **缺点**：**有前提条件！**
        -   相关矩阵 $R$ 必须是**可逆的**（非奇异的）。
        -   如果存在*完全共线性*（$R_j^2 = 1$），$R$ 将不可逆。
        -   如果 $p \ge n$（变量数多于观测数），$R$ 也不可逆。
        -   如果矩阵“病态”（ill-conditioned，即接近不可逆），$R^{-1}$ 的计算会充满数值误差。

`check_collinearity_vif` 的策略是：**默认尝试方法B（快速），如果检测到任何问题，则自动回退到方法A（慢速）。**

### R 源码

```{r}
check_collinearity_vif <- function(data, cols = NULL, threshold = 10) {
  # 1. 输入校验和预处理 (与 check_collinearity_cor 类似)
  stopifnot(is.data.frame(data))
  rcond_tol <- .Machine$double.eps^0.5 # 用于检测"病态矩阵"的极小阈值

  # (解析 cols ...)
  if (is.null(cols)) {
    cols_to_use <- names(data)
  } else if (is.numeric(cols)) {
    cols_to_use <- names(data)[cols]
  } else {
    cols_to_use <- as.character(cols)
  }
  if (any(!cols_to_use %in% names(data))) {
    stop("cols 中有不存在 data 的列名")
  }

  if (length(cols_to_use) == 0) {
    warning("没有指定任何列。")
    return(data.frame(predictor = character(0), vif = numeric(0), is_high = logical(0)))
  }

  sub_data <- data[cols_to_use]

  # (筛选数值型...)
  is_num <- vapply(sub_data, is.numeric, logical(1))
  valid_cols_initial <- names(is_num)[is_num]
  excluded_non_numeric <- names(is_num)[!is_num]
  if (length(excluded_non_numeric) > 0) {
    warning("已排除非数值型列: ", paste(excluded_non_numeric, collapse = ", "))
  }
  if (length(valid_cols_initial) == 0) {
    warning("所有指定的列都不是数值型，无法计算 VIF。")
    return(data.frame(predictor = character(0), vif = numeric(0), is_high = logical(0)))
  }

  pred_df <- sub_data[valid_cols_initial]

  # (处理缺失值...)
  rows_ok <- stats::complete.cases(pred_df)
  pred_df_complete <- pred_df[rows_ok, , drop = FALSE]
  n_obs <- nrow(pred_df_complete)

  # 初始化VIF向量，默认为 NA
  vif_values <- setNames(
    rep(NA_real_, length(valid_cols_initial)),
    valid_cols_initial
  )

  if (n_obs < 2) {
    warning(paste("完整观测行数 (n_obs =", n_obs, ") 不足 2，无法计算 VIF。"))
  } else {
    # (处理零方差列...)
    sds <- vapply(pred_df_complete, sd, numeric(1), na.rm = FALSE)
    is_zero_var <- (sds == 0) | is.na(sds)
    excluded_zero_var <- names(is_zero_var)[is_zero_var]
    valid_cols_final <- names(is_zero_var)[!is_zero_var]

    if (length(excluded_zero_var) > 0) {
      warning("已排除零方差/常数列 (VIF 设为 NA): ", paste(excluded_zero_var, collapse = ", "))
    }

    p <- length(valid_cols_final)

    if (p == 0) {
      warning("所有数值型列（在完整行上）都是常数或NA，VIF 均设为 NA。")
    } else if (p == 1) {
      # 只有一个变量，没有共线性，VIF=1
      vif_values[valid_cols_final] <- 1.0
    } else {
      # 2. 核心：尝试 "方法B" (快速)，并设置回退标志
      use_lm_method <- FALSE

      # 2a. 回退检查 1: p >= n
      if (p >= n_obs) {
        warning(paste("p (有效自变量数 =", p, ") >= n (观测行数 =", n_obs, ")。相关矩阵奇异，将回退到 lm 辅助回归法。"))
        use_lm_method <- TRUE
      }

      if (!use_lm_method) {
        R <- stats::cor(pred_df_complete[valid_cols_final]) # nolint

        # 2b. 回退检查 2: 矩阵是否正定 (Positive Definite)
        R_chol <- try(chol(R), silent = TRUE) # nolint

        if (inherits(R_chol, "try-error")) {
          warning("相关矩阵非正定，自动回退到逐变量辅助回归法。")
          use_lm_method <- TRUE
        } else {
          # 2c. 回退检查 3: 矩阵是否 "病态" (Ill-Conditioned)
          r_cond <- rcond(R)

          if (r_cond < rcond_tol) {
            warning(sprintf("相关矩阵病态 (rcond = %g < %g)，自动回退到逐变量辅助回归法。", r_cond, rcond_tol))
            use_lm_method <- TRUE
          } else {
            # 2d. 成功！使用 "方法B"
            inv_r <- chol2inv(R_chol)
            vif_values[valid_cols_final] <- diag(inv_r)
          }
        }
      }

      # 3. 核心：执行 "方法A" (慢速回退)
      if (use_lm_method) {
        # 准备好设计矩阵 (加一列截距项，便于 lm.fit)
        X_all_with_intercept <- cbind( # nolint
          `(Intercept)` = 1,
          as.matrix(pred_df_complete[valid_cols_final])
        )

        for (i in seq_len(p)) {
          this_pred_name <- valid_cols_final[i]

          # 准备 y (即 X_j) 和 X (即其他变量)
          y <- X_all_with_intercept[, i + 1, drop = FALSE]
          X_matrix <- X_all_with_intercept[, -(i + 1), drop = FALSE] # nolint

          # 使用底层的 lm.fit (比 lm() 快)
          fit <- try(lm.fit(x = X_matrix, y = y), silent = TRUE)

          if (inherits(fit, "try-error")) {
            r2 <- NA_real_
          } else {
            # 手动计算 R-squared
            rss <- sum(fit$residuals^2)
            # 注意：TSS (Total Sum of Squares) 的计算
            tss <- sum(y^2) - sum(y)^2 / n_obs

            if (abs(tss) < .Machine$double.eps) {
              # 如果 y 是常数 (本应被零方差检查捕获，但作为双重保险)
              r2 <- NA_real_
            } else {
              r2 <- 1 - rss / tss
            }

            # 修复浮点数精度问题 (e.g., 1.0000000000000002)
            if (!is.na(r2) && r2 > 1 && (r2 - 1) < .Machine$double.eps) {
              r2 <- 1.0
            }
          }

          # 3b. 从 R^2 计算 VIF
          if (is.na(r2)) {
            vif_val <- NA_real_
          } else if (abs(1 - r2) < .Machine$double.eps) {
            # R^2 = 1 (完全共线性)
            vif_val <- Inf
          } else {
            vif_val <- 1 / (1 - r2)
          }
          vif_values[this_pred_name] <- vif_val
        }
      }
    }
  }

  # 4. 格式化输出
  vif_table <- data.frame(
    predictor = names(vif_values),
    vif = vif_values,
    is_high = !is.na(vif_values) & vif_values >= threshold
  )
  rownames(vif_table) <- NULL

  # 按VIF值降序排列，NA值放在最后
  vif_table <- vif_table[
    order(vif_table$vif, decreasing = TRUE, na.last = TRUE),
  ]

  return(vif_table)
}
```

### 步骤详解

1.  **预处理 (1)**：

-   与 `check_collinearity_cor` 几乎相同：筛选数值列、`complete.cases`、排除零方差列。
-   `vif_values` 被初始化为 `NA`。对于被排除的非数值列或零方差列，它们最终的VIF将保持为 `NA`。
-   **特殊情况**：如果 $p=1$（只有一个有效变量），`VIF` 直接设为 1。

2.  **核心逻辑 - 尝试方法B (2)**：

-   `use_lm_method` 标志：这是一个“开关”，默认为 `FALSE`（使用方法B），如果任何检查失败，则设为 `TRUE`（切换到方法A）。
-   **(2a) 检查 `p >= n`**：这是最常见的奇点（Singularity）来源。如果变量多于观测，矩阵 $X'X$（相关矩阵 $R$ 与此相关）是不可逆的。必须回退。
-   **(2b) 检查正定性**：`R_chol <- try(chol(R), ...)`。`chol` 是 **Cholesky 分解**，是计算矩阵逆的**第一步**，也是检查矩阵是否为**正定矩阵**（Positive Definite）的最快方法。如果 `chol` 失败（例如，由于浮点数不精确导致对角线出现非正值），`try` 会捕获错误，我们必须回退。
-   **(2c) 检查病态**：`r_cond <- rcond(R)`。`rcond` 计算**“条件数（Condition Number）的倒数”**。
    -   **条件数**是衡量“矩阵接近不可逆”的指标。一个大的条件数（`rcond` 相应地会很小）意味着矩阵是**“病态”**的。
    -   对病态矩阵求逆，会导致极大的数值误差。例如，$1 / (1 - 0.9999999999999999)$ 应该是一个大数，但 $1 / (1 - 1.0000000000000001)$ 却是一个负数！
    -   `rcond_tol` 是一个常用的阈值（约 $1.49 \times 10^{-8}$）。如果 `rcond(R)` 小于此值，我们认为求逆是不安全的，必须回退。
-   **(2d) 成功**：如果所有检查都通过，`inv_r <- chol2inv(R_chol)`（从Cholesky分解高效计算逆）并 `diag(inv_r)`（提取对角线）一步到位。

1.  **核心逻辑 - 执行方法A (3)**：

-   `if (use_lm_method)`：如果任何一个检查失败，此开关为 `TRUE`，执行此代码块。
-   `for (i in seq_len(p))`：对*每一个*变量 $X_j$ (即 `this_pred_name`) 执行循环。
-   `y <- ...` 和 `X_matrix <- ...`：巧妙地从总矩阵中提取出 $X_j$ 作为 `y`，提取出 $X_{-j}$（所有其他变量）作为 `X_matrix`。
-   `fit <- try(lm.fit(...), ...)`：
    -   `lm.fit` 是R中执行回归的底层函数。它跳过了 `lm()` 函数中涉及的公式解析、数据框转换等耗时步骤，直接在**矩阵** `x` 和**向量** `y` 上进行计算，速度快得多。
-   **手动计算** $R^2$：
    -   `rss <- sum(fit$residuals^2)`：残差平方和（Residual Sum of Squares）。
    -   `tss <- sum(y^2) - sum(y)^2 / n_obs`：总平方和（Total Sum of Squares）。
    -   `r2 <- 1 - rss / tss`：$R^2$ 的标准定义。
-   **VIF 计算**：
    -   `if (abs(1 - r2) < ...)`：这是对 $R^2 = 1$ 的稳健检查（考虑到浮点精度）。
    -   `vif_val <- Inf`：如果 $R^2 = 1$，则 $1 - R^2 = 0$，VIF 为无穷大（Inf）。
    -   `vif_val <- 1 / (1 - r2)`：标准公式。

4.  **格式化输出 (4)**：

-   将 `vif_values` 向量转换为一个整洁的 `data.frame`。
-   `order(vif_table$vif, ...)`：一个非常有用的排序，将VIF值从高到低排列，并将所有 `NA` 值（来自非数值型或零方差列）放在最后。

让我们创建一个更"邪恶"的数据集，包含完美共线性。

-   `x1`, `x2`, `x3` 存在完美共线性（$x3 = x1 + x2$）。这导致它们的辅助回归 $R^2 = 1$，因此 VIF 算出来是 `Inf`（无穷大）。
-   `x4` 和 `x5` 存在极高的相关性，但不是**完美**共线性。这导致它们的 VIF 值非常高（远超 10），但不是 `Inf`。
-   函数正确地识别了所有 VIF 超过 10 的变量，并将 `is_high` 设为 `TRUE`。

## 结论

我们深入分析了两种诊断共线性的R函数：

1.  **`check_collinearity_cor`**：

-   **做什么**：通过贪心算法自动*移除*变量。
-   **原理**：计算一个“总相关负担” (`cached_sum`)，并迭代地移除“负担”最重的变量，同时高效地更新其邻居的“负担”。
-   **适用场景**：适用于特征工程和预筛选，当你希望快速减少特征数量并消除**成对**高相关性时。

2.  **`check_collinearity_vif`**：

-   **做什么**：计算每个变量的VIF值，用于*诊断*。
-   **原理**：实现了两种方法。首选 $VIFs = \text{diag}(R^{-1})$（快速，矩阵求逆），但在检测到数值不稳定（`p >= n`、非正定、病态）时，自动回退到 $VIF_j = 1 / (1 - R_j^2)$（慢速，辅助回归）。
-   **适用场景**：**这是标准的共线性诊断方法**。它比相关矩阵法更强大，因为它可以检测到一个变量被*多个*其他变量组合解释的情况（例如，$X_1 \approx 0.5 X_2 + 0.5 X_3$，即使 $X_1$ 与 $X_2$、$X_3$ 的两两相关性都不高）。
